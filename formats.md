## Standardized formats for brain activity data

Please find a list of commonly used data formats. Notice something missing? Please [contribute](./contribute){:target="_blank" rel="noopener noreferrer"}.

| # | Format | Description | Citation | Modality |
| :--- | :--- | :--- | :--- | :--- |
| 1 | **[BRAINformat](https://bitbucket.org/oruebel/brainformat/){:target="_blank" rel="noopener noreferrer"}** | The BRAINformat library aims to create a general framework for standardization of scientific data. It defines application-independent design concepts and modules, is open source, and utilizes the HDF5 data format. | [Rübel et al., 2016](https://www.biorxiv.org/content/10.1101/024521v1){:target="_blank" rel="noopener noreferrer"} | Electrophysiology |
| 2 | **[KWIK](https://github.com/kwikteam/klusta){:target="_blank" rel="noopener noreferrer"}** | KWIK files were developed for the Klusta suite spike sorting software to contain the results of the spike sorting session. Based on HDF5, these files include information about the spike times and spike clusters, as well as all the metadata. | [Kadir et al., 2014](https://doi.org/10.1162/NECO_a_00661){:target="_blank" rel="noopener noreferrer"}; [Rossant et al. 2016](https://doi.org/10.1038/nn.4268){:target="_blank" rel="noopener noreferrer"} | Electrophysiology |
| 3 | **[NEO](https://neo.readthedocs.io/){:target="_blank" rel="noopener noreferrer"}** | NEO succeeded the Neuroshare API, which provided a design specification for core neuroscience data objects. NEO similarly aims to decouple the data representation step in analysis pipelines to create a unified format facilitating downstream analysis in software pipelines, packages, and frameworks. | [Garcia et al., 2014](https://doi.org/10.3389/fninf.2014.00010){:target="_blank" rel="noopener noreferrer"} | Electrophysiology |
| 4 | **[ONE](https://docs.internationalbrainlab.org/){:target="_blank" rel="noopener noreferrer"}** | Developed to standardize data sharing among multiple laboratories involved in the International Brain Laboratory (IBL) consortium, ONE defines a set of conventions for naming and organizing data files. Because ONE utilizes a ‘object.attribute.extension’ naming scheme, it can be used across file formats. | [International Brain Laboratory et al., 2023](http://dx.doi.org/10.1038/s41586-025-09226-1){:target="_blank" rel="noopener noreferrer"} | Electrophysiology, Optical imaging |
| 5 | **[NWB](https://nwb.org/){:target="_blank" rel="noopener noreferrer"}** | NWB provides a common standard to share, archive, use, and build analysis tools for diverse neurophysiology data. Based on the HDF5 format, NWB was designed to support data from intracellular and extracellular electrophysiology, optical imaging, as well as behavioral data. | [Teeters et al., 2015](https://doi.org/10.1016/j.neuron.2015.10.025){:target="_blank" rel="noopener noreferrer"}; [Rubel et al., 2022](https://doi.org/10.7554/eLife.78362){:target="_blank" rel="noopener noreferrer"} | Electrophysiology, Optical imaging |
| 6 | **[Orca](http://crcns.org/files/data/nwb/h1/NWBh1_09_Keith_Godfrey.pdf){:target="_blank" rel="noopener noreferrer"}** | Orca (also called BORG) was developed by the Allen Institute for Brain Science and designed to store electrophysiology and optical imaging data. It is based on the HDF5 format. | [Godfrey, 2014](http://crcns.org/files/data/nwb/h1/NWBh1_09_Keith_Godfrey.pdf); [Friedsam, 2016](https://dash.harvard.edu/server/api/core/bitstreams/7312037e-6fe1-6bd4-e053-0100007fdf3b/content){:target="_blank" rel="noopener noreferrer"} | Electrophysiology, Optical imaging |
| 7 | **[NIX](https://g-node.github.io/nix/){:target="_blank" rel="noopener noreferrer"}** | The NIX file format was designed to store diverse neurophysiology data, including electrophysiology and optical imaging, as well as behavioral data. Based on HDF5, NIX aims to provide a flexible and robust standard for organizing and sharing complex neuroscience datasets in a hierarchical and extensible way. | [Martone, 2020](https://f1000research.com/documents/9-358){:target="_blank" rel="noopener noreferrer"}; [Stoewer et al. 2014](https://doi.org/10.3389/conf.fninf.2014.18.00027) | Electrophysiology, Optical imaging |
| 8 | **[EDF, EDF+](https://www.edfplus.info/){:target="_blank" rel="noopener noreferrer"}** | An early standardized data format, EDF was used for the storage of sleep recordings and included EEG, respiration, body temperature, EMG, and other related data. With EDF+, the format was expanded to provide additional flexibility. | [Kemp et al., 1992](https://doi.org/10.1016/0013-4694(92)90009-7){:target="_blank" rel="noopener noreferrer"}; [Kemp & Olivan, 2003](https://doi.org/10.1016/S1388-2457(03)00123-8){:target="_blank" rel="noopener noreferrer"} | EEG |
| 9 | **[MEF](https://github.com/msel-source/meflib?tab=readme-ov-file){:target="_blank" rel="noopener noreferrer"}** | MEF provides open source software to read, write, and process large volumes of neural data. It utilizes range encoding to provide a high degree of data compression, a 128-bit encryption system for securely storing patient information, and a redundancy check to verify the integrity of the compressed data. | [Brinkmann et al., 2009](https://doi.org/10.1016/j.jneumeth.2009.03.022){:target="_blank" rel="noopener noreferrer"} | EEG, Electrophysiology |
| 10 | **[BIDS](https://bids.neuroimaging.io/){:target="_blank" rel="noopener noreferrer"}** | BIDS is a standard for organizing and describing neuroimaging and EEG data to promote reproducibility and interoperability. It defines a structured format to arrange files and metadata in order to facilitate the processing and analysis of datasets across tools and the sharing across labs. | [Gorgolewski et al., 2016](https://doi.org/10.1038/sdata.2016.44){:target="_blank" rel="noopener noreferrer"} | EEG, Neuroimaging |
| 11 | **[NIFTI](https://en.wikipedia.org/wiki/Neuroimaging_Informatics_Technology_Initiative){:target="_blank" rel="noopener noreferrer"}** | The NIFTI file format is designed to simplify the storage of neuroimaging data. It packages both image data and its associated metadata into a single file to facilitate analysis and sharing. | [Cox et al., 2004](https://www.nitrc.org/docman/view.php/26/204/TheNIfTI){:target="_blank" rel="noopener noreferrer"} | Neuroimaging |
| 12 | **[DICOM](https://www.dicomstandard.org/){:target="_blank" rel="noopener noreferrer"}** | DICOM is an international standard for medical imaging. It defines the formats for medical images that can be exchanged with the data and quality necessary for clinical use. This format is widely used in healthcare (X-ray, CT, MRI, ultrasound, etc.) as well as in neuroimaging. | [NEMA, 1993](https://www.dicomstandard.org/){:target="_blank" rel="noopener noreferrer"} | Neuroimaging, Other |
